# Installation

XLMTools installs in two ways: as an **MCP server** (every major agent client supports it) or as a **standalone CLI** (for terminal use). Pick whichever fits your workflow — or install both.

## Install the MCP server

XLMTools works natively with 12+ MCP clients. Pick yours:

<Tabs items={['Claude Code', 'Claude Desktop', 'Cursor', 'Windsurf', 'VS Code', 'Gemini CLI', 'Codex', 'Zed', 'Continue', 'Cline', 'Goose']}>

<Tabs.Tab>
```bash
claude mcp add xlmtools npx @xlmtools/cli
```

Claude Code downloads and runs the server automatically on next session.
</Tabs.Tab>

<Tabs.Tab>
Open your config file:
- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`
- Linux: `~/.config/Claude/claude_desktop_config.json`

```json
{
  "mcpServers": {
    "xlmtools": {
      "command": "npx",
      "args": ["-y", "@xlmtools/cli"]
    }
  }
}
```

Fully quit and restart Claude Desktop.
</Tabs.Tab>

<Tabs.Tab>
Create `.cursor/mcp.json` (project) or `~/.cursor/mcp.json` (global):

```json
{
  "mcpServers": {
    "xlmtools": {
      "command": "npx",
      "args": ["-y", "@xlmtools/cli"]
    }
  }
}
```

Or use the UI: Settings → Tools & MCP → Add new MCP server.
</Tabs.Tab>

<Tabs.Tab>
Edit `~/.codeium/windsurf/mcp_config.json`:

```json
{
  "mcpServers": {
    "xlmtools": {
      "command": "npx",
      "args": ["-y", "@xlmtools/cli"]
    }
  }
}
```
</Tabs.Tab>

<Tabs.Tab>
VS Code Copilot uses a different schema — note `servers` key (not `mcpServers`). Create `.vscode/mcp.json`:

```json
{
  "servers": {

      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@xlmtools/cli"]
    }
  }
}
```

Copilot Chat must be in Agent mode for MCP tools to appear.
</Tabs.Tab>


## Install the standalone CLI

If you want to use XLMTools directly from your terminal (no MCP host required):

```bash
npm install -g @xlmtools/cli
```

This gives you two binaries:
- `xlmtools` — the MCP server (for clients above)
- `xlm` — the standalone CLI

```bash
xlm wallet
xlm crypto bitcoin,stellar
xlm search "Stellar hackathon" --count 3
```

See the [Standalone CLI guide](/guides/cli) for all commands.

## Install the Agent Skill

XLMTools also ships as a cross-client Agent Skill that teaches your agent when and how to use its tools. Works natively in Claude Code, Cursor 2.4+, Windsurf, VS Code Copilot, Codex CLI, Gemini CLI, Goose, and Cline.

<Tabs items={['Prompt', 'pnpm', 'npx', 'bunx']}>

<Tabs.Tab>
Paste this into any agent:

```
Read https://xlmtools.com/skill.md and follow the instructions to install XLMTools.
```

The agent fetches the skill and installs itself.
</Tabs.Tab>

<Tabs.Tab>
```bash
pnpm dlx skills add github:Blockchain-Oracle/xlmtools --skill xlmtools
```
</Tabs.Tab>

<Tabs.Tab>
```bash
npx skills add github:Blockchain-Oracle/xlmtools --skill xlmtools
```
</Tabs.Tab>

<Tabs.Tab>
```bash
bunx skills add github:Blockchain-Oracle/xlmtools --skill xlmtools
```
</Tabs.Tab>

</Tabs>

See the [Agent Skill guide](/guides/skill) for details.

## Verify installation

After installing, ask your agent:

```
What tools does XLMTools provide?
```

The agent will call the `tools` tool and list all 21 available tools with their prices.

<Callout type="info">
On first run, XLMTools generates a Stellar testnet keypair at `~/.xlmtools/config.json` and auto-funds it with XLM via friendbot. The wallet is used to sign USDC micropayments for paid tool calls. You'll still need to grab testnet USDC from [faucet.circle.com](https://faucet.circle.com) to use paid tools.
</Callout>
