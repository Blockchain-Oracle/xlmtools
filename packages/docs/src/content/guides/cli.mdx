# Standalone CLI

XLMTools also ships as a standalone command-line tool. Use it directly from your terminal without needing an MCP host. Same wallet, same payments, same tools.

## Install

```bash
npm install -g @xlmtools/cli
```

This installs two binaries:
- `xlmtools` — the MCP stdio server (used by Claude, Cursor, etc.)
- `xlm` — the standalone terminal CLI

## Usage

```bash
xlm <tool> [args] [--flag value]
```

### Free tools

```bash
xlm wallet
xlm tools
xlm crypto bitcoin,ethereum,stellar
xlm weather Lagos
xlm domain xlmtools-mcp.xyz
xlm dex-orderbook XLM/USDC --limit 5
xlm oracle-price BTC
xlm stellar-account GDJIFRCNKCYINRP3FCPGEOWB4L3JBU3LN4Y2AZNXCV7CK7MEGZTNG5NG
```

### Paid tools

Paid tools automatically handle the MPP payment — you just need testnet USDC in your wallet.

```bash
xlm search "Stellar MPP micropayments" --count 5
xlm research "Soroban smart contracts" --num-results 3
xlm stocks AAPL
xlm screenshot https://stellar.org --format png
xlm scrape https://stellar.org
xlm image "a pulsar star in deep space" --size 1024x1024
xlm youtube "Stellar development"
```

Every paid response prints a receipt line showing the Stellar transaction hash:

```
  Payment: $0.003 USDC · tx/8f3a1b2c4d5e... · stellar testnet
```

## How it differs from the MCP server

| Feature | MCP server (`xlmtools`) | Standalone CLI (`xlm`) |
| --- | --- | --- |
| Usage | Via Claude, Cursor, Windsurf | Direct terminal commands |
| Interface | stdin/stdout JSON-RPC | `process.argv` parsing |
| Persistence | Long-running process | One-shot (exits after each command) |
| Budget tracking | Yes (session-scoped) | No (resets per invocation) |
| Response caching | Yes (5-min TTL) | No (each call is fresh) |
| Wallet | Shared (`~/.xlmtools/config.json`) | Shared (same file) |
| Payments | MPP via mppx | MPP via mppx (same flow) |

Both modes share the same Stellar keypair and API endpoint. You can install once and use both.
