# MCP Host Setup

XLMTools works with any MCP-compatible host. Here is how to install it in each supported client.

All clients use the same npm package: `@xlmtools/mcp`.

<Callout type="info">
The `-y` flag on `npx` auto-confirms the install prompt. Without it, `npx` will ask for confirmation on first run.
</Callout>

## Claude Code

One command, no config file needed:

```bash
claude mcp add xlmtools npx @xlmtools/mcp
```

Optional scope flags:
- `-s user` — install globally for your user
- `-s project` — write to shared `.mcp.json` for your team

Verify:
```bash
claude mcp list
```

## Claude Desktop

Config file location:
- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`
- Linux: `~/.config/Claude/claude_desktop_config.json`

Or via the app: Claude menu → Settings → Developer → Edit Config.

```json
{
  "mcpServers": {
    "xlmtools": {
      "command": "npx",
      "args": ["-y", "@xlmtools/mcp"]
    }
  }
}
```

Fully quit and restart Claude Desktop after saving.

## Cursor

Config file:
- Global: `~/.cursor/mcp.json`
- Per-project: `.cursor/mcp.json` in your workspace root

Or via UI: Settings → Tools & MCP → Add new MCP server.

```json
{
  "mcpServers": {
    "xlmtools": {
      "command": "npx",
      "args": ["-y", "@xlmtools/mcp"]
    }
  }
}
```

A green dot in the Tools & MCP panel means XLMTools is running.

## Windsurf

Config file:
- macOS/Linux: `~/.codeium/windsurf/mcp_config.json`
- Windows: `%USERPROFILE%\.codeium\windsurf\mcp_config.json`

Or via UI: Cascade panel → MCPs icon (top-right) → Configure.

```json
{
  "mcpServers": {
    "xlmtools": {
      "command": "npx",
      "args": ["-y", "@xlmtools/mcp"]
    }
  }
}
```

## VS Code (GitHub Copilot)

VS Code Copilot has native MCP support. Note the root key is `servers`, not `mcpServers`.

Config: `.vscode/mcp.json` in your workspace, or via Command Palette → `MCP: Open User Configuration`.

```json
{
  "servers": {
    "xlmtools": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@xlmtools/mcp"]
    }
  }
}
```

Copilot Chat must be in **Agent mode** for MCP tools to appear.

## Cline (VS Code extension)

In VS Code, click the MCP Servers icon in Cline's top nav → Installed → Configure MCP Servers.

```json
{
  "mcpServers": {
    "xlmtools": {
      "command": "npx",
      "args": ["-y", "@xlmtools/mcp"],
      "disabled": false,
      "autoApprove": []
    }
  }
}
```

## Zed

Config file:
- macOS/Linux: `~/.config/zed/settings.json`
- Windows: `%APPDATA%\Zed\settings.json`

Zed uses a different schema — note `context_servers` and `source: "custom"`:

```json
{
  "context_servers": {
    "xlmtools": {
      "source": "custom",
      "command": "npx",
      "args": ["-y", "@xlmtools/mcp"],
      "env": {}
    }
  }
}
```

## Continue (VS Code / JetBrains)

Create `.continue/mcpServers/xlmtools.yaml` in your workspace:

```yaml
name: XLMTools MCP
version: 0.0.1
schema: v1
mcpServers:
  - name: XLMTools
    type: stdio
    command: npx
    args:
      - "-y"
      - "@xlmtools/mcp"
```

<Callout type="warning">
MCP only works in Continue's **Agent mode**, not Chat or Edit modes.
</Callout>

## Goose (Block)

Easiest: run the interactive configurator:

```bash
goose configure
```

Then choose: Add Extension → Command-line Extension (stdio). Fill in:
- Name: `xlmtools`
- Command: `npx -y @xlmtools/mcp`
- Timeout: `300`

Or edit the config directly at `~/.config/goose/config.yaml`:

```yaml
extensions:
  xlmtools:
    name: xlmtools
    type: stdio
    enabled: true
    cmd: npx
    args:
      - "-y"
      - "@xlmtools/mcp"
    timeout: 300
```

## 5ire

Use the Tools page in the app: Tools → Add Server. Fill in:
- Command: `npx`
- Args: `-y @xlmtools/mcp`

## Roo Code (VS Code extension)

Roo Code → MCP Servers → Edit Global MCP, or create `.roo/mcp.json` in your workspace:

```json
{
  "mcpServers": {
    "xlmtools": {
      "command": "npx",
      "args": ["-y", "@xlmtools/mcp"],
      "disabled": false,
      "alwaysAllow": [],
      "timeout": 60
    }
  }
}
```

## LibreChat

Add to your `librechat.yaml`:

```yaml
mcpServers:
  xlmtools:
    type: stdio
    command: npx
    args:
      - "-y"
      - "@xlmtools/mcp"
```

Restart LibreChat for the server to load.

## OpenClaw

[OpenClaw](https://openclaw.ai) is an open-source AI agent that runs across WhatsApp, Telegram, Slack, Discord, and 20+ messaging platforms. It supports MCP servers via stdio.

Config: `~/.openclaw/openclaw.json`

```json
{
  "mcpServers": {
    "xlmtools": {
      "command": "npx",
      "args": ["-y", "@xlmtools/mcp"]
    }
  }
}
```

Or via the OpenClaw CLI:

```bash
openclaw mcp set xlmtools '{"command":"npx","args":["-y","@xlmtools/mcp"]}'
```

Restart the OpenClaw gateway after adding the server.

## Config schemas at a glance

Three schemas exist across the ecosystem:

| Schema | Root key | Used by |
| --- | --- | --- |
| Standard MCP | `mcpServers` | Claude Desktop, Claude Code, Cursor, Windsurf, Cline, Roo Code, 5ire, LibreChat, OpenClaw |
| VS Code Copilot | `servers` + `type: "stdio"` | VS Code GitHub Copilot |
| Zed | `context_servers` + `source: "custom"` | Zed |
| Continue YAML | `mcpServers` array + `name`/`type` | Continue |
| Goose YAML | `extensions` + `cmd` (not `command`) | Goose |

## Verifying the connection

After setup, ask your agent:

```
List all available XLMTools tools
```

The agent should call the `tools` tool and return a list of all 21 XLMTools tools with their prices.

## Troubleshooting

**Command not found: npx** — Install Node.js 22+ from [nodejs.org](https://nodejs.org).

**Server starts but no tools appear** — Update your MCP host to the latest version. Some clients require Agent mode (Continue, VS Code Copilot).

**Wallet errors** — On first run, XLMTools creates a wallet at `~/.xlmtools/config.json`. If corrupted, delete it:
```bash
rm ~/.xlmtools/config.json
```
A new wallet will be generated and auto-funded on testnet.
