# Agent Skill

PULSAR also ships as an Agent Skill. A skill is a `SKILL.md` file that teaches Claude (or any skill-capable agent) when and how to use a tool, without consuming context until it is needed.

## Skill vs MCP server — what's the difference?

PULSAR is both, and they are complementary.

| | MCP Server | Skill |
| --- | --- | --- |
| What it ships | Tool schemas (machine-readable) | Procedural instructions (markdown) |
| Context cost | Always in context | ~100 tokens at startup, full body loads on demand |
| Who invokes | Agent calls tools directly | Agent reads instructions, then calls tools |
| Install | `claude mcp add pulsar npx @pulsar/mcp` | `~/.claude/skills/pulsar/SKILL.md` |

The skill's job is to teach your agent:
- When PULSAR is the right choice
- The decision tree mapping user intent to tool
- Paid tool warnings and cost awareness
- How to read payment receipts
- CLI fallback when MCP tools aren't available

## Three install methods

### 1. Via the community skills CLI

Uses [`vercel-labs/skills`](https://github.com/vercel-labs/skills), the emerging standard for skill distribution:

```bash
pnpm dlx skills add github:Blockchain-Oracle/pulsar --skill pulsar

# or with npm / bun
npx skills add github:Blockchain-Oracle/pulsar --skill pulsar
bunx skills add github:Blockchain-Oracle/pulsar --skill pulsar
```

This clones our skills directory and installs `SKILL.md` into the right location for whichever agents you have installed.

### 2. Via prompt

Paste this sentence into any agent:

```
Read https://pulsar.tools/skill.md and follow the instructions to install PULSAR.
```

The agent fetches the file over HTTPS, reads the instructions, and installs the skill itself. No CLI required.

### 3. Manually

Download `SKILL.md` from our `/skill` page, then save it to:

```
~/.claude/skills/pulsar/SKILL.md
```

Restart or reload your agent and the skill becomes available.

<Callout type="info">
For Claude Code, skills are live-loaded — you do not need to restart after adding a new skill. Just save the file and it becomes available on the next tool call.
</Callout>

## What's in the skill

The SKILL.md contains:

- **YAML frontmatter** with a "pushy" description that lists trigger keywords (crypto, stock, weather, Stellar, etc.)
- **Mode priority** — use MCP tools first, fall back to `pulsar-cli`
- **Decision tree** — user intent to tool lookup table
- **Tool catalog** — all 21 tools with prices and parameters
- **CLI invocation examples**
- **Payment receipt format**
- **Budget management guidance**
- **Safety checklist** for paid calls
- **Troubleshooting**

## Why skills matter

Skills solve the "I have 21 tools, which one do I use?" problem. Without a skill, the agent has to infer from tool names and descriptions alone. With a skill, the agent gets a decision tree explicitly telling it "when the user asks about stock prices, use `stocks`."

This is especially important for:
- Distinguishing free vs paid tools
- Knowing which tool is best for ambiguous requests ("tell me about X" — `search` or `research`?)
- Ensuring the agent surfaces payment receipts to the user
- Preventing accidental expensive calls (image generation is $0.040, research is $0.010)

<Callout type="warning">
Skills currently tend to under-trigger in Claude. PULSAR's skill description is intentionally "pushy" with trigger keywords to counteract this — it explicitly lists every topic the skill covers so Claude auto-invokes it reliably.
</Callout>
