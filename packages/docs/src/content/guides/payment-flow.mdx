# Payment Flow

XLMTools uses the **Micropayment Protocol (MPP)** to handle pay-per-call billing on the Stellar network. This guide explains the full payment lifecycle for a paid tool call.

## Overview

MPP is a challenge-response protocol. When the XLMTools API server receives a request for a paid tool without a valid payment proof, it responds with HTTP `402 Payment Required` and includes payment instructions in the response headers. The XLMTools CLI client (`mppx`) handles the rest automatically.

## Step-by-step flow

### 1. Tool call initiated

Your AI agent calls a paid tool (e.g. `search`). The XLMTools CLI sends the request to the API server.

### 2. Server returns 402

The API server checks if the request includes a valid MPP payment proof. On the first attempt, it won't, so the server responds:

```http
HTTP/1.1 402 Payment Required
X-MPP-Version: 1
X-MPP-Amount: 0.003
X-MPP-Asset: USDC
X-MPP-Destination: GDEST...
X-MPP-Network: testnet
X-MPP-Memo: search:abc123
```

### 3. `mppx` builds and signs payment

`mppx` (the 402 orchestration layer) reads the response headers and invokes the configured `@stellar/mpp` charge method to build a **Soroban SAC USDC transfer**:

- **From**: Your local wallet address
- **To**: The server's `STELLAR_RECIPIENT` address
- **Amount**: The tool's price (e.g. 0.003 USDC)
- **Asset**: `USDC_SAC_TESTNET` — the Soroban USDC contract on Stellar testnet

The transaction is signed with your local Stellar secret key (stored in `~/.xlmtools/config.json`). **Your key never leaves your machine** — only the signed proof is transmitted.

### 4. Request retried with proof

`mppx` attaches the signed transaction proof to the retry request headers and re-sends the original request automatically. The agent sees none of this.

### 5. Server verifies and executes

The API's `mppx.charge()` middleware (backed by `@stellar/mpp/charge/server`):

1. Verifies the MPP proof signature
2. Submits the Soroban transaction to Stellar testnet
3. Confirms on-chain settlement
4. Executes the tool call against the backend (Brave, Exa, OpenAI, etc.)
5. Returns the results plus a Stellar transaction hash

The entire flow happens in a few seconds.

## Security

- **Local signing** — Your secret key never leaves your machine.
- **Per-call payments** — Each tool call is a separate, auditable transaction.
- **On-chain verification** — Every payment is recorded on the Stellar ledger and can be independently verified.
- **No pre-authorization** — XLMTools never holds or pools your funds. You pay exactly per call.
