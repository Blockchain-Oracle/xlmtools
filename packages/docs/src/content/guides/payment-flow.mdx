# Payment Flow

PULSAR uses the **Micropayment Protocol (MPP)** to handle pay-per-call billing on the Stellar network. This guide explains the full payment lifecycle for a paid tool call.

## Overview

MPP is a challenge-response protocol. When the PULSAR API server receives a request for a paid tool without a valid payment proof, it responds with HTTP `402 Payment Required` and includes payment instructions in the response headers. The PULSAR CLI client (`mppx`) handles the rest automatically.

## Step-by-step flow

### 1. Tool call initiated

Your AI agent calls a paid tool (e.g. `search`). The PULSAR CLI sends the request to the API server.

### 2. Server returns 402

The API server checks if the request includes a valid MPP payment proof. On the first attempt, it won't, so the server responds:

```http
HTTP/1.1 402 Payment Required
X-MPP-Version: 1
X-MPP-Amount: 0.003
X-MPP-Asset: USDC
X-MPP-Destination: GDEST...
X-MPP-Network: testnet
X-MPP-Memo: search:abc123
```

### 3. MPP client builds payment

The `mppx` library reads the 402 headers and builds a Soroban SAC (Stellar Asset Contract) USDC transfer transaction:

- **From**: Your local wallet address
- **To**: The server's payment destination
- **Amount**: The tool's price (e.g. 0.003 USDC)
- **Memo**: Server-provided memo for matching

### 4. Transaction signed locally

The transaction is signed with your local Stellar secret key (stored in `~/.pulsar/config.json`). Your key never leaves your machine.

### 5. Request retried with proof

The signed transaction is attached to the request as an MPP payment proof header, and the request is retried.

### 6. Server verifies and executes

The API server:

1. Verifies the payment proof signature
2. Submits the transaction to the Stellar network
3. Confirms settlement
4. Executes the tool call
5. Returns the results

The entire flow happens in a few seconds.

<Callout type="warning">

</Callout>

## Security

- **Local signing** — Your secret key never leaves your machine.
- **Per-call payments** — Each tool call is a separate, auditable transaction.
- **On-chain verification** — Every payment is recorded on the Stellar ledger and can be independently verified.
- **No pre-authorization** — PULSAR never holds or pools your funds. You pay exactly per call.
